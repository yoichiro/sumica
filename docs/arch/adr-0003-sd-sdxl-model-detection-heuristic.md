# ADR 3: SD/SDXLモデルの判別に名前ヒューリスティックフィルタを採用する

## Context

LoRAやHires.fixなど一部の設定はSDXL系モデルとSD1.5系モデルで挙動や適用可否が異なるため、サーバー側でどちらの系統かを判別する必要がありました。「Stable DiffusionのモデルなのかXLのモデルなのか区別はつきますか？」という問いを受けて調査した結果、AUTOMATIC1111/Forgeの`/sdapi/v1/sd-models`は`title`/`hash`/`filename`しか返さず、アーキテクチャの種別を示すメタデータを持たないことが分かりました。信頼性の高い判定を行うには`.safetensors`のヘッダーを直接読み、デュアルテキストエンコーダーや2048次元のcross-attentionの有無を確認する必要があり、実装コストが高くなります。

## Decision

高コストな`.safetensors`ヘッダー解析ではなく、チェックポイント名に`"xl"`という文字列が含まれるかどうかで判定する、低コストな名前ヒューリスティックフィルタを採用します。

## Status

置き換え済み（[[adr-0009-safetensors-header-sdxl-detection]]により置き換え）。名前ヒューリスティックの誤判定リスクを解消するため、`.safetensors`ヘッダーを直接読む方式に移行しました。

## Consequences

- 実装が単純で、SDモデル一覧のプロキシ処理に軽量なフィルタを足すだけで済みました。
- 命名規則に反するファイル名（`"xl"`を含まないSDXLモデル、あるいは`"xl"`を含むSD1.5モデル）が存在した場合は誤判定します。この限界は既知のトレードオフとして受け入れています。
