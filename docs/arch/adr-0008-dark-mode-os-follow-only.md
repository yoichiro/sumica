# ADR 8: ダークモードはOS設定への追従のみとし、手動切り替えは持たない

## Context

ダークモード対応を検討するにあたり、実装案として「OS設定に自動追従するのみ」というシンプルな案と、「OS自動追従に加えて☀️/🌙の手動切り替えボタンとlocalStorageへの状態永続化を持つ」ハイブリッド案が比較されました。後者はより一般的なダークモード実装として提案されたものでした。

## Decision

あえて最も軽量な案を採用し、手動切り替えボタンや状態の永続化は持たず、CSSの`prefers-color-scheme`のみでダークモードを実現します。配色は木炭色（チャコール）ベースのパレットを採用し、深緑・漆黒（OLED）ベースのパレット案は不採用としました。実装は`:root`（ライト）とダークモード用の上書きにCSS変数を分割する形で行いました。

## Status

承認済み

## Consequences

- 状態管理やlocalStorageへの永続化ロジックが一切不要になり、実装・保守コストを最小化できました。
- 一方で、OSの設定とは独立してアプリ内だけダークモードを切り替えたいユーザーのニーズには応えられません。将来そのような要望が出た場合は、この決定を見直す必要があります。
