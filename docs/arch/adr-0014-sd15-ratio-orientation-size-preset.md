# ADR 14: SD1.5にもアスペクト比+向き+サイズのプリセット式ピッカーを導入する

## Context

[[adr-0010-sdxl-ratio-orientation-size-preset]]でSDXLの解像度UIを「幅と高さの独立選択」から「アスペクト比+向き+サイズ」のプリセット式に切り替えたとき、SD1.5は**意図的にスコープ外**にしていました。理由はADR 10のContextに書いた通り、SD1.5はバケット学習をしていないので「比率」という抽象化が純粋に見た目の便利さでしかなく、SDXLほど嬉しさが大きくない、というものでした。

しかし実際に両モードを行き来してみると、以下の摩擦が明らかになりました。

- **UI構造の非対称**: SDXLは比率チップ、SD1.5は幅/高さドロップダウンで、頭の切り替えが必要。
- **SD1.5で選べる比率が9通りに固定**: 3×3の直積の中でも実質的に4つの比率ファミリー（1:1, 4:3, 3:2, 2:1）しかカバーされていない。**5:4, 5:3, 16:9** といったよく使う比率は、既存の {512, 768, 1024} だけでは表現できないため事実上生成できない状態だった。
- **バッチ生成のUIも別構造**: SDXLは「比率×向き×サイズ」の3軸、SD1.5は「幅×高さ」の2軸で、洋一郎さんが両方を使い分けるたびに小さな認知負荷が発生していた。

洋一郎さんから「SD1.5もSDXLと同じUIにしたい」という要望があり、具体的な設計は次の観点で詰めました。

- **既存の 512/768/1024 のピクセル値をすべて到達可能にする** — SD1.5で慣れた解像度を捨てない。
- **新規追加する比率は SD1.5 実運用の安全圏を優先** — 総ピクセル数 0.25〜0.5 MP、両辺 384〜768 に収まる範囲。5:4 (640×512, 0.33MP), 5:3 (640×384, 0.25MP) はネイティブ級。16:9 (1024×576, 0.59MP) は片辺 1024 のボーダーライン品質だが実用需要が大きいので採用。
- **21:9 や 3:1 のような極端な比率は SDXL 専用の売り**として残す — SD1.5では構図が破綻しやすく、SDXLとSD1.5のUIを差別化する意味でもSD1.5には入れない。
- **サイズ (S/M/L) の扱いはSD1.5に合わせて非対称にする** — 1:1 は S(512²)/M(768²)/L(1024²) の3段階を持てるが、非1:1のSD1.5では M より小さいと粗くなり大きいと破綻が増えるので、**Mだけの単一サイズ**にする。SDXL版は7比率すべてがS/M/L対応だったのに対し、SD1.5は明示的にこの制約を持つ。

## Decision

SD1.5選択時のメインフォーム解像度ピッカーと、バッチ生成の「サイズの組合せ」モードを、SDXLと同じ「比率+向き+サイズ」の3段階UIに切り替えます。

- 定数`SD15_PRESETS`（`client/src/App.tsx`）で7つのアスペクト比を定義：`1:1`, `5:4`, `4:3`, `3:2`, `5:3`, `16:9`, `2:1`。
- **1:1のみ S/M/L の3サイズを持つ**：512×512（native）/ 768×768 / 1024×1024。他の6ratioは M サイズ1つだけ：
  - `5:4 M` → 640×512（新規）
  - `4:3 M` → 1024×768（既存）
  - `3:2 M` → 768×512（既存）
  - `5:3 M` → 640×384（新規）
  - `16:9 M` → 1024×576（新規）
  - `2:1 M` → 1024×512（既存）
- **型定義**: `Sd15Ratio` と `Sd15Preset` を新規追加。`Sd15Preset.sizes` は `M` を必須、`S`/`L` はオプショナル（1:1のみ全部埋まる）。`SdxlSize`（`'S' | 'M' | 'L'`）と `SdxlOrientation`（`'landscape' | 'portrait' | 'square'`）はアーキ中立な列挙型として両方で共有。
- **リゾルバとファインダ**: `resolveSd15Dimensions()`（非正方形ではM固定・向きで W↔H スワップ）と `findSd15Selection()`（現在の width/height から picker 座標を逆引き）を SDXL 版と同じ形で追加。
- **UI**:
  - メインフォーム: 7ratioチップ → 向きトグル（1:1では非表示）→ サイズトグル（1:1のみ表示、S/M/L）→ 実寸ラベル。
  - バッチダイアログ: 比率マルチ選択 × 向きマルチ選択 × サイズマルチ選択 の3軸。1:1は選択サイズごとに1ジョブ、非1:1は選択向きごとに1ジョブ（サイズ選択は無視、M固定）。デデュープ付き。
- **state**: `selectedSd15Ratio`, `selectedSd15Orientation`, `selectedSd15Size` の3つと、バッチ用の `selectedSd15BatchRatios`, `selectedSd15BatchOrientations`, `selectedSd15BatchSizes` の3セット。SDXL側の同名state（Sd15なし）と完全に平行構造。
- **useEffect**:
  - アーキテクチャ切替時の分岐で、SD1.5選択なら `findSd15Selection(width, height)` で picker を逆引き。マッチしなければ `1:1 S` (512×512) にフォールバック。
  - Picker（`selectedSd15Ratio/Orientation/Size`）変化時に `resolveSd15Dimensions()` で width/height を更新する新規 effect を追加。
- **`loadIntoForm`**: SDXL側と同じく、`arch === 'sd15'` のときも `findSd15Selection(item.width, item.height)` で picker を同期。同一アーキテクチャ内のロード（例: SD1.5→SD1.5 のロード）でも picker チップが正しく更新されるよう明示同期。
- **削除するもの**: `SD15_SIZE_OPTIONS` 定数、`selectedWidths` / `selectedHeights` state、`handleSwapDimensions` ハンドラ、`toggleSize` ヘルパー、`ArrowLeftRight` アイコンimport。すべて旧SD1.5用のドロップダウンUIの残骸で、新ピッカーでは使わない。
- **データモデルは無変更**: `{ width, height }` を保持する`BatchJob`、生成メタデータ、`GenerationRecord`、Firestore/ローカル永続化は一切変更なし。プリセットは UI 層でのみ意味を持ち、生成パイプラインから見れば単なる (W, H) のタプル。

## Status

承認済み

- [[adr-0010-sdxl-ratio-orientation-size-preset]]のDecision節にある「SD1.5の解像度UI・バッチのサイズモードは従来通り（`SD15_SIZE_OPTIONS = [512, 768, 1024]`の幅×高さ独立選択）を維持します。SD1.5はバケット学習ではないため、比率抽象化のメリットが薄いと判断しました」の記述を、本ADRが supersede します。「バケット学習ではないので比率化のメリットが薄い」の判断は、実運用してみたら**UI一貫性のメリットのほうが大きかった**という学びに更新されています。

## Consequences

- SD1.5とSDXLでフォーム操作の頭の切り替えが不要になりました。両方のアーキテクチャで「比率チップを選ぶ → 向きを選ぶ → 必要ならサイズを選ぶ」の同じ手順。
- SD1.5で新規に**5:4, 5:3, 16:9** の3比率が到達可能になりました。既存9通り + 新規6通り（3ratio × 2向き）= 15通りが基本セット。従来の {512,768,1024} 由来の9通りはすべて保持されている。
- **非1:1のサイズは意図的にMだけ**: SDXLとの明示的な差別化。SD1.5でLサイズ（1.4MP相当）を狙うと破綻確率が高いこと、SサイズもSDXLと違って学習分布から外れることが理由。将来「SD1.5でもLサイズが欲しい」需要が明確になれば、Hires.fixとの併用前提で追加可能な設計は残っている。
- **極端な比率（21:9, 3:1）はSDXL専用**: SD1.5では構図破綻が多いので入れないという判断が、`SD15_PRESETS`と`SDXL_PRESETS`の中身の差として明文化されました。SDXLへの動機付けとしても機能。
- 既存のSD1.5生成メタデータは`{ width, height }`しか持たないため、`findSd15Selection()`が既知の9つの寸法から6つ（5:4, 5:3, 16:9の3ratio × 2向き）を除いた**9通りに逆引きヒット**します。過去データは全て`findSd15Selection`でpicker化でき、「フォームにロード」の挙動は継続。
- バッチ生成の「サイズの組合せ」モードは、SDXLと同じ3軸クロス積UIに統一。SDXL版で撤廃した`MAX_SIZE_COMBINATIONS`のキャップ撤廃（[[adr-0012-remove-batch-size-combination-cap]]）はそのまま維持され、SD1.5でも意図的に大量ジョブを詰めることができます。
- `handleSwapDimensions`（幅⇄高さ入替ボタン）が撤廃されました。新UIでは「向き」トグルが同じ役割を果たします。
- SDXL側の`ratioIsBucket`/`isSdxlBucket`（⭐バッジ）はSD1.5には無い：SD1.5にバケット学習の概念がないため、UI上の視覚差別化としてバッジ無しでフラットに扱う。
