# ADR 48: ライトボックス機能を「画像拡大表示のモーダルオーバーレイ」として新設する

## Context

履歴ギャラリーのサムネイル、およびメインプレビューに表示される画像を拡大表示するための UI が必要になりました。設計方針として次の 2 通りが考えられました。

- (A) 別ページ (別ルート) に遷移し、その画面で画像を大きく表示する
- (B) 現在のページの上にモーダルオーバーレイとして拡大画像を表示する (ライトボックス方式)

Sumica はシングルページアプリケーションとして構築されており、明示的なルーティングライブラリを持っていません (CLAUDE.md でも「No router」と明記)。また、拡大表示はギャラリーや生成結果を眺めながら短時間で切り替える操作であり、遷移コストが低いほうが好まれます。

## Decision

私たちは (B) を採用します。ライトボックスは React state で管理するモーダルオーバーレイとして実装します。

- `lightboxUrl` state (画像 URL または null) を持ち、非 null のときフルビューポートを覆う position: fixed のオーバーレイを描画します。
- 閉じる操作は **Esc キー** と **背景クリック** の両方をサポートします。
- z-index は既存の `DeleteConfirmModal` より高く設定し、削除確認ダイアログを開いた状態からもライトボックスを閉じる操作が優先されるようにします。
- 拡大ボタン (虫めがねアイコン) はサムネイルおよびメインプレビュー画像の **右下** に配置します。当初は右上を提案しましたが、既存のサムネイル右上には保存先バッジ (☁️ Firebase / 📁 ローカル) と選択チェックバッジ (✓) が既に配置されており衝突するため、右下に決定しました。
- **ズーム倍率変更・パン操作は明示的にスコープ外** とし、YAGNI (You Aren't Gonna Need It) の判断で見送りました。ライトボックスは「元画像の等倍または縮小フィット表示」までを扱います。

(A) を却下した主な理由は次の通りです。第一に、React Router 等の導入が必要になり、SPA の状態管理と URL の整合を保つコストが発生します。第二に、ブラウザの戻る操作でライトボックスを閉じる挙動は一見自然に見えますが、実際にはギャラリーのスクロール位置や選択状態の復元と組み合わせるとバグが起きやすく、モーダルの明示的な閉じる操作 (Esc/背景クリック) のほうが単純で堅牢です。

## Status

承認済み

## Consequences

- ライトボックスはこれ以降の多くの機能拡張の土台となりました。[[adr-0013-lightbox-info-panel]] (情報パネル)、[[adr-0023-lightbox-random-shuffle]] (ランダムシャッフル)、[[adr-0025-lightbox-view-transitions]] (View Transitions アニメーション)、[[adr-0026-lightbox-fullscreen-overlay]] (フルスクリーン昇格)、[[adr-0027-lightbox-space-selection-toggle]] (Space 選択トグル)、[[adr-0033-lightbox-eye-button-open-in-preview]] (Eye ボタン)、[[adr-0040-lightbox-slideshow-and-random-mode-toggle]] および [[adr-0041-slideshow-interval-selection-and-countdown-ring]] (スライドショー) はいずれもこの基盤の上に構築されました。
- ライトボックスの状態は React state のみで管理し、URL や履歴 API (`history.pushState`) を触らないため、ブラウザの「戻る」ボタンで表示中のライトボックスは閉じません。代わりにギャラリー画面から抜けてしまうため、ユーザーに違和感を与える可能性があります。実用上は Esc キーが定着しているため問題視されていませんが、モバイルブラウザでは戻るジェスチャーとの衝突が起きる可能性があります (Sumica はデスクトップ主体のため優先度は低い)。
- ズーム/パンをスコープ外にしたことにより、ライトボックスの実装が単純に保たれ、後続の機能追加が短期間で行えました。将来的に画像の細部を確認したいニーズが強くなれば追加検討の余地はありますが、[[adr-0026-lightbox-fullscreen-overlay]] のフルスクリーン化で暫定的な代替になっています。
- 拡大ボタン右下配置は、既存バッジとの衝突回避の副産物として、サムネイルの視覚重心が右下に偏る効果を持ちました。これは特に問題視されていませんが、後日 [[adr-0018-gallery-caption-static-display]] でキャプションを追加した際に「右下ボタンとキャプションの視覚バランス」を意識する必要が生じました。
