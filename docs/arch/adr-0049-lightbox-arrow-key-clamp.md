# ADR 49: ライトボックスの方向キー ←/→ による前後送りにクランプ方式を採用する

## Context

[[adr-0048-lightbox-overlay-foundation]] で導入したライトボックスを表示中に、方向キー ←/→ で前後の画像に切り替える機能を追加することにしました。前後の対象は、ギャラリー画面の日付フィルターを適用した後の表示順 (`filteredHistory`) に沿います。

前後送りの端 (最初の画像で ← を押した / 最後の画像で → を押した) の扱いには次の 2 通りが考えられました。

- (A) 端で止まる (クランプ)
- (B) 端でループ (最後の次は最初へ、最初の前は最後へ)

## Decision

私たちは (A) クランプ方式を採用します。端に達した状態でさらに同じ方向のキーを押しても、何も起こりません (画像は切り替わらず、現在のまま)。

(B) を却下した主な理由は次の通りです。第一に、ループ方式ではユーザーが「今どこにいるか」を見失いやすく、特に長い履歴一覧では周回してしまっても気づかないことがあります。第二に、クランプ方式は「もう次はない」という状態が UI 上で明確に伝わります (何度キーを押しても画像が変わらない)。第三に、Sumica の履歴は時系列に意味があり (新しい生成が先頭)、順序に端があること自体が情報として意味を持ちます。

## Status

承認済み

## Consequences

- 単純明快な UX が得られ、ユーザーがライトボックス内で「もう次がない」ことに気づけます。
- [[adr-0026-lightbox-fullscreen-overlay]] の全画面表示は既存のライトボックスキーリスナーをそのまま流用する設計 ([[adr-0048-lightbox-overlay-foundation]] の再利用) だったため、クランプ挙動が全画面表示にも自動的に適用されました。追加実装は不要でした。
- 副作用として、ユーザーが履歴の先頭または末尾に到達したことを明示するフィードバック (端に到達した瞬間の視覚エフェクトなど) は持ちません。「押しても動かない = 端」という間接的なフィードバックのみです。実運用ではこれで十分機能していますが、より明示的なフィードバックを望む場合の拡張余地は残っています。
- 後続の [[adr-0040-lightbox-slideshow-and-random-mode-toggle]] のスライドショー機能でも、同じクランプ方針を継承しています (最後の画像に到達したらスライドショーは停止し、先頭に戻りません)。
