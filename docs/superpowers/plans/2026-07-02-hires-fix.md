# Hires.fix Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Hires.fix (Stable Diffusion's high-resolution second pass) support — a toggle plus upscaler/scale/steps/denoising controls — that flows through single and batch generation, persists in metadata, and restores via "♻️ フォームにロード".

**Architecture:** Two tasks in sequence — server first (add `enableHr`/`hrUpscaler`/`hrScale`/`hrSecondPassSteps`/`denoisingStrength` to `generateImage()`, the `/api/generate` route, `GenerationMetadata`, and a new `/api/sd-upscalers` endpoint), then client (state, fetch, UI block, request wiring, preview display, `loadIntoForm`). Mirrors the existing sampler/scheduler pattern exactly. Batch generation needs no changes — it reads the same form state as single generation.

**Tech Stack:** Express 5 + TypeScript ESM (server, tsx); React 19 + TypeScript + Vite 8 (client); oxlint; no new dependencies.

## Global Constraints

- Modify `server/index.ts`, `client/src/App.tsx`, and `client/src/firebase.ts` only.
- `npm run typecheck --prefix server` must exit 0 after Task 1.
- `cd client && npx tsc -b` and `npm run lint --prefix client` must exit 0 after Task 2.
- Comments in English only.
- No new npm dependencies.
- Only the 5 basic Hires.fix fields (`enableHr`, `hrUpscaler`, `hrScale`, `hrSecondPassSteps`, `denoisingStrength`) — no `hr_sampler_name`/`hr_scheduler`/`hr_prompt`/`hr_negative_prompt`/`hr_resize_x/y`, no per-batch-job override.
- Spec: `docs/superpowers/specs/2026-07-02-hires-fix-design.md`.

- [ ] **Step 4: Add the `GET /api/sd-upscalers` endpoint**

Insert immediately after the closing `});` of the `/api/sd-loras` endpoint, before the `// 8. Delete selected generations` comment:

```ts
// 7c. List Stable Diffusion upscalers (for the Hires.fix upscaler picker). Merges
// GAN-based upscalers with latent-space upscale modes into one flat name list,
// since SD's `hr_upscaler` field accepts either kind interchangeably.
app.get('/api/sd-upscalers', async (_req: Request, res: Response) => {
  try {
    const [upscalersRes, latentRes] = await Promise.all([
      axios.get(`${stableDiffusionUrl}/sdapi/v1/upscalers`, { timeout: 5000 }),
      axios.get(`${stableDiffusionUrl}/sdapi/v1/latent-upscale-modes`, { timeout: 5000 }),
    ]);
    const names = (data: unknown): string[] =>
      Array.isArray(data)
        ? data.map((u: { name?: string }) => u.name).filter((n): n is string => Boolean(n))
        : [];
    const upscalers = [...names(upscalersRes.data), ...names(latentRes.data)];
    res.json({ upscalers });
  } catch (error) {
    console.error('Failed to fetch SD upscalers:', (error as Error).message);
    res.json({ upscalers: [] });
  }
});
```
