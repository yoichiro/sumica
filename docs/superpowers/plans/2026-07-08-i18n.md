# UI Internationalization (JP/EN) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add English UI alongside the existing Japanese UI. Locale is resolved once at load time from `?hl=ja` / `?hl=en` URL query params, then falls back to `navigator.language` (starts with `ja` → `ja`, otherwise `en`).

**Architecture:** A custom lightweight `client/src/i18n/` module with three files: `index.ts` (locale resolver + `t` export), `ja.ts` (Japanese strings organized by feature area), and `en.ts` (English strings with `typeof ja` type annotation to enforce structural equality). No external i18n library. Server code is untouched — server-returned error messages are used verbatim as the `details` interpolation argument in the client's translated wrapper messages.

**Tech Stack:** TypeScript, React 19, Vite, Vitest. No new npm dependencies.

## Global Constraints

- Client is ESM (`"type": "module"`); use `import`, not `require`.
- No new npm dependencies — the i18n module is custom, no library.
- Do NOT change server code (`server/*.ts`), Firebase schema, or `GenerationData` shape.
- Locale resolution order is fixed: `?hl=ja` → `?hl=en` → `navigator.language.toLowerCase().startsWith('ja')` → `en` fallback. `hl` values other than `ja` / `en` are ignored and fall through.
- `document.documentElement.lang` is set to the resolved locale (side effect at module load).
- `en.ts` MUST use `export const en: typeof ja = { ... }` to enforce structural equality via TypeScript.
- Interpolation format: static strings are plain strings; strings with variables are functions `(arg: type) => \`template\``.
- Do NOT translate: `console.log`/`console.error` messages, code comments, `docs/*` documentation, `CLAUDE.md`, `SDXL_PRESETS` `'S'`/`'M'`/`'L'`/`'1:1'`/`'landscape'` etc. (symbolic constants), `index.html`'s `<title>` (already English).
- Commit messages must be one-line English; MUST NOT contain `Claude`, `Co-Authored-By`, or any AI-assistance trailer.

- [ ] **Step 4: Create the index.ts module**

Create `client/src/i18n/index.ts` with:

```ts
import { ja } from './ja';
import { en } from './en';

export type Locale = 'ja' | 'en';

// Resolve the UI locale once at module load. Priority: URL query ?hl=ja/en
// override, then navigator.language starting with "ja" → ja, otherwise en.
// Invalid hl values (e.g. ?hl=fr) fall through to navigator.language, so
// a mistyped URL never breaks the app.
export function resolveLocale(): Locale {
  const hl = new URLSearchParams(window.location.search).get('hl');
  if (hl === 'ja') return 'ja';
  if (hl === 'en') return 'en';
  return navigator.language.toLowerCase().startsWith('ja') ? 'ja' : 'en';
}

export const locale: Locale = resolveLocale();
export const t = locale === 'ja' ? ja : en;

// Set the HTML lang attribute so browser spell-check, screen readers, and
// CSS :lang() selectors align with the actual UI language.
if (typeof document !== 'undefined') {
  document.documentElement.lang = locale;
}
```

- [ ] **Step 2: Write the ADR file**

Create `docs/arch/adr-0020-i18n-jp-en-support.md` with the four required sections in exact order (Context / Decision / Status / Consequences), all in です・ます 体 Japanese. Base the content on the spec doc `docs/superpowers/specs/2026-07-08-i18n-design.md`, condensing to focus on:

- **Context**: what forced the decision — the UI was Japanese-only, blocking non-Japanese speakers; ~176 strings across 13 files. Requirement was JA/EN with URL param override + browser language fallback.
- **Decision**: custom lightweight `client/src/i18n/` module, no external library, load-time locale resolution, `typeof ja` type constraint on `en`, server messages stay English and are wrapped in translated messages on the client. - **Status**: 承認済み.
- **Consequences**: benefits (no runtime dependency, 100 % typed dictionary, simple mental model), trade-offs (no live locale switching, no plural rules or number formatting — YAGNI for now), extension path (a UI toggle + localStorage can be added without touching the dictionary contract).

Refer to related ADRs: `[[adr-0007-gallery-thumbnail-strategy]]`, `[[adr-0015-ui-component-split-with-hybrid-state]]`, and the two spec files.
